# Monster Hardware Explorer v1.0 Sprint 4

## 這一版做什麼

Sprint 4 將已實機驗證的硬幣機與100元收鈔器整合成「不找零現金付款控制器」。輸入應付金額後，程式會累計實收金額、顯示尚差金額，剛好達標就同時停止收幣與收鈔。

本版沒有開放 `/dev/ttyS3` 出鈔、硬幣出幣、支付命令、Reset 或 Escrow。

## 安全收款流程

1. 開始前先傳送 Coin Disable 與 Bill Disable，清除上一個收款狀態。
2. 重新讀取 Coin Setup 與 Bill Setup，兩者校驗成功才進入收款。
3. 只接受實機驗證過的1、5、10、50元硬幣與100元紙鈔。
4. 依「尚差金額」動態產生硬幣接受遮罩；手動出幣位元永遠為0。
5. 尚差少於100元時，立即停用100元紙鈔。
6. 實收剛好等於應付時，依最後收到的現金種類優先停用該模組，再停用另一個模組。
7. 120秒逾時、取消、App進入背景、ttyS1中斷或異常時，均停止兩種現金入口。

## 取消付款的重要限制

這版沒有找零或退款硬體控制。如果取消、逾時或異常時已經收到現金，程式會：

- 保留實收金額與各面額數量。
- 顯示紅色「需要人工處理」狀態。
- 阻止開始下一筆付款。
- 等操作人員人工退款或完成記帳後，才可按「確認已人工處理」清除警示。

## 第一次實機整合測試

1. 關閉原廠 Lite QC Tool。
2. 用 Android Studio 安裝 Sprint 4。
3. 選擇 `ttyS1 · MDB（硬幣＋收鈔）`、`9600`，按「連線」。
4. 應付金額輸入 `110`，按「開始付款（120秒）」。
5. 等畫面顯示可收硬幣與100元紙鈔後，投入一張100元。
6. 確認畫面變成「已付100、尚差10」，而100元紙鈔已停用。
7. 再投入一枚10元。
8. 確認畫面顯示綠色「付款完成：應付110／已付110／尚差0」。
9. 匯出 Session ZIP，上傳 `payment_summary.txt`、`session.log` 與原始 MDB 記錄供核對。

每次只投入一枚硬幣或一張紙鈔，等畫面金額更新後再投入下一枚。第一次測試不要使用500元、1000元、20元或任何找零功能。

## 匯出新增內容

`payment_summary.txt` 會記錄：

- 付款狀態
- 應付、已付、尚差與超收金額
- 硬幣枚數、紙鈔張數與面額明細
- 是否有待人工處理款項
- 最後硬體控制狀態
