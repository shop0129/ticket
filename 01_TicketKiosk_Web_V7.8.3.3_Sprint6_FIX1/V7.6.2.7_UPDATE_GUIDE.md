# V7.6.2.7 票券入場、襪子領取與順序修正

## 覆蓋方式
直接以更新包內檔案覆蓋原專案相同路徑，再重新部署 Firebase Hosting。

## 本版修正
- 票券上移／下移後以 `sortOrder` 儲存，重新整理或雲端同步後順序仍保留。
- 新增票券設定「可否入場」。
- 新增票券設定「櫃台領取商品 → 至櫃台領取襪子」。
- 訂單會保留 `canEnter`、`admissionRequired`、`pickupItem`。
- 可入場票會建立驗票實例；不可入場商品不會誤列為等待入場。
- 售票完成畫面會顯示「🧦 至櫃台領取襪子 × 數量」。

## Firebase
不需要更新 Rules 或 Functions。票券設定會沿用原本的票券雲端同步。

## 測試建議
1. 新增「兒童襪子」，分類選其他票種。
2. 可否入場選「可以入場」。
3. 櫃台領取商品選「至櫃台領取襪子」。
4. 儲存後將票券上移，再重新整理確認順序。
5. 購買一張，確認完成頁顯示入場手環與襪子領取。
6. 員工後台確認訂單可正常入場。
