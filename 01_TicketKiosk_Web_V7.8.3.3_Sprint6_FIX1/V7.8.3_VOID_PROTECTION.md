# V7.8.3 Void Protection

## 修正重點

訂單作廢後，電腦後台與手機員工後台會共同鎖定以下權益操作：

- 領取玩具
- 轉換玩具點數
- 設定等待領取
- 作廢訂單補綁會員後再轉點
- 驗票與入場（沿用 Ticket Status Engine）

## 多層防護

1. 畫面層：已作廢訂單的相關按鈕會變灰並停用。
2. 操作層：即使直接呼叫函式，也會顯示「此訂單已作廢」。
3. Firebase Transaction 層：領取或轉點寫入前，會重新讀取最新訂單狀態；若其他裝置剛完成作廢，交易不會提交。
4. 訂單作廢時同步寫入 `status / playStatus / validationStatus / cancelled / voided`，避免不同頁面使用不同狀態欄位而漏判。

## 已轉點後再作廢

電腦後台原有會員回滾流程會檢查點數是否足夠，足夠時扣回該訂單轉入的玩具點數；若點數已被使用，系統會阻止直接作廢並要求店長先調整點數。
