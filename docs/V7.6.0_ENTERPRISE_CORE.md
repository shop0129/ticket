# V7.6.0 Enterprise Core

## 核心原則

票券實例為狀態判斷的主要來源。當訂單有 `ticketInstances` 時，系統會依票券狀態推導訂單狀態，避免 `playStatus` 與驗票卡不同步。

## 狀態

- `waiting`：等待入場
- `playing`：遊玩中
- `finished`：已離場
- `cancelled`：已作廢
- `not_required`：不需入場

## 相容欄位

每次狀態轉換會同步寫入：

- `playStatus`
- `validationStatus`
- `ticketStatus`
- `ticketInstances[].status`

## 部署

本版不需修改 Firebase Rules。覆蓋檔案後重新部署，並在手機或平板重新整理一次；Service Worker 會清除舊快取。
