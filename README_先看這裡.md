# 小怪獸售票機 V7.8.3.3 Sprint 6 FIX1 完整版

這是以 ticket_V7.8.3.3_Point_Form_Reset_Fix_FULL.zip 為正式底檔，重新製作的 Sprint 6 現金付款整合版。

## 這次要更新什麼

你已經成功安裝 Android 現金控制器並拿到配對碼 23360700，所以目前不必重裝 Android App。

正式上線只需要把資料夾 01_TicketKiosk_Web_V7.8.3.3_Sprint6_FIX1 裡的全部檔案，完整覆蓋到售票機 GitHub Pages 專案根目錄，再部署。

不要再上傳舊的 01_TicketKiosk_Web_V7.3_Sprint6，否則會把 V7.8.3.3 功能退回舊版。

## 第一次安全測試

1. 先開啟已安裝的 Monster Hardware Explorer。
2. 確認已連線 ttyS1／9600，畫面仍顯示本機橋接與配對碼。
3. 部署新版網頁後，先完全關閉點餐機上的售票機 PWA，再重新開啟；FIX1 會自動接管並重載新版。
4. 先看右下角，必須只顯示一個 `V7.8.3.3 S6 FIX1 · ONLINE`。如果仍看到 `V7.8.1` 或 `V7.4.0`，不可投入現金。
5. 選擇一筆實付剛好100元、沒有使用會員點數的訂單。
6. 按「現金付款」。
7. 第一次輸入配對碼 23360700。
8. 確認畫面顯示應付 NT$100，控制器也顯示同一筆100元訂單。
9. 只投入一張100元紙鈔。
10. 確認售票機只建立一筆訂單、只出票一次，Staff／店長後台可看到該訂單。
11. 從 Android 控制器匯出最新 Session ZIP，再上傳分析。

第一次測試不要投入硬幣、500元、1000元，也不要測找零。

## 現金安全限制

- 只接受已實機驗證的1／5／10／50元硬幣與100元紙鈔。
- 不找零；尚差金額不足100元時會停用100元入口。
- 20元、500元、1000元、出幣、出鈔、Reset、Escrow及未知MDB指令仍封鎖。
- 部分收款後取消、逾時或中斷會鎖定人工處理，不會自動退款或出票。
- 付款授權、訂單與出票都具有防重複處理。
